### AI 스코어링 가중치 공식 (F-1.3)

```
total_score = skill_score(0~100)      * 0.4
            + reliability_score(0~100) * 0.3
            + performance_score(0~100) * 0.3
            + bonus_score(0~10)          ← 상한 10점 고정 (total 100 초과 방지)
```

**콜드 스타트 보정**: `performance_score = 0`인 신규 인력(`is_new_talent = TRUE`)은  
매칭 쿼리에서 별도 가중치(`NEW_TALENT_BOOST`)를 적용하여 첫 프로젝트 매칭 기회 보장

```
신규 인력 유효 점수 = total_score + NEW_TALENT_BOOST(기본 10점)
                    — 단, 3회 이상 프로젝트 수행 후 자동 해제
```

| 기능 ID | 기능명 | 상세 설명 | IN / OUT | 우선순위 |

| F-2.2 | AI 지능형 인력 추천 | 프로젝트 요건과 가용 인력 간 벡터 유사도 분석 → 상위 인력 추천 (TOP 5) | IN: 프로젝트 요건 / OUT: 추천 인력 리스트·추천 사유 | 상 |

| F-2.4 | AI 후보자 이력 비교 | 다수 후보자 기술스택·평점·이력·예상 비용 병렬 비교 분석 | IN: 후보자 선택 리스트 / OUT: 비교 테이블·AI 리포트 | 중 |
| F-2.5 | 심층 인터뷰 가이드 생성 | F-1.9 이상 징후 기반 → PM용 기술 질문 및 검증 질문 리스트 자동 생성 (`match_proposals.interview_guide`) | IN: 사전 평가 결과 / OUT: 맞춤형 인터뷰 질문지 | 중 |

### AI 매칭 로직 (UC-2.3 상세)

```
1. PM이 프로젝트 요구사항 입력 (필수 기술, 기간, 예산 범위)
2. 1차 필터링:
   - availability_status = 'AVAILABLE'
   - desired_rate BETWEEN budget_min AND budget_max  ← 예산 필터 (신규)
   - work_type 일치 여부 (REMOTE/ONSITE/HYBRID)      ← 근무형태 필터 (신규)
3. requirement_embedding ↔ profile_embedding 코사인 유사도 계산
4. AI 포인트 추가 계산:
   - 과거 선호 업무 유형 분석
   - 예상 매칭 성공 확률 계산
   - rejected_by_pm = TRUE 인력 하위 노출               ← PM 거절 이력 반영 (신규)
   - is_new_talent = TRUE 인력 보정 가중치 적용          ← 콜드스타트 보정 (신규)
5. total_score + 유사도 점수 합산 정렬 → TOP N 추천
```

### 챗 시나리오 예시

```
[Talent - 모바일]
사용자: "이번 주 업무 보고 써줘. React 컴포넌트 설계 하고 코드 리뷰했어"
   AI: "아래 형식으로 초안을 작성했어요. 수정 후 제출하시겠어요?

        [주간 업무 보고 - 2026년 4월 3주차]
        ■ 주요 업무
          - React 컴포넌트 아키텍처 설계 완료

        ■ 이슈: 없음"

   AI: "업무 보고가 제출되었습니다."
```
