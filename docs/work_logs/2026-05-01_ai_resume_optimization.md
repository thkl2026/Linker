# 2026-05-01 AI 이력서 분석 엔진 최적화 및 신뢰성 확보

## 1. 개요
이력서 분석 과정에서 발생하는 데이터 누락, 한글 깨짐, 화면 분석 실패 문제를 해결하고, 분석 결과의 추적성을 확보하기 위해 시스템 전반을 재설계함.

## 2. 주요 문제 및 해결 내역

### 2.1 데이터 휘발성 문제 (Audit Log 도입)
- **문제**: 분석 실패 시 AI가 어떤 데이터를 반환했는지 확인할 방법이 없어 원인 파악 불가.
- **해결**: `resume_analysis_logs` 테이블을 생성하고, 분석 결과(JSON)를 영구 저장하도록 구현.
- **성과**: 모든 분석 결과의 물리적 증거 확보 및 실시간 디버깅 가능.

### 2.2 한글 인코딩 및 날짜 표준화
- **문제**: Windows 서버 환경에서 한글이 `???`로 깨짐, AI가 날짜를 비표준 형식(24.01 등)으로 응답하여 시스템 바인딩 실패.
- **해결**:
    - RestTemplate에 UTF-8 MessageConverter 강제 적용.
    - `YYYY-MM-DD` 표준 날짜 정규화 로직(formatDate) 구현.
    - "현재", "재직중" 등 특수 문자열을 시스템 날짜로 자동 변환.

### 2.3 네트워크 타임아웃 장애 (Timeout 10s -> 180s)
- **문제**: AI 분석 소요 시간(평균 30~60초)이 프론트엔드 Axios 타임아웃(10초)을 초과하여 화면에서 강제 실패 처리됨.
- **해결**: 프론트엔드 타임아웃 설정을 3분으로 연장하여 안정적인 분석 결과 수신 보장.

### 2.4 데이터 바인딩 및 런타임 에러 방지
- **문제**: 필수값(이름, 시작일 등) 누락 시 프론트엔드 폼에서 에러 발생.
- **해결**:
    - 이름 추출 실패 시 정규표현식 기반 Fallback 추출 로직 추가.
    - 학력 시작일 누락 시 종료일 기준 3년 전으로 자동 생성.
    - 프론트엔드 폼 바인딩 시 Optional Chaining 적용으로 Crash 방지.

## 3. 디버깅 및 유지보수 도구
- **DbQueryTool**: 서버 구동 없이 DB에 직접 접속하여 최신 분석 JSON을 덤프하는 Java 유틸리티.
- **@PostConstruct Logs**: 서버 시작 시 최근 분석 로그 1건을 자동으로 콘솔에 출력.
- **Gradle Task**: `./gradlew dbQuery` 명령어로 간편하게 DB 데이터 확인 가능.

## 4. 향후 과제
- 분석된 학력/경력 리스트를 전문가 프로필 테이블로 일괄 저장하는 기능 고도화.
- 대량의 이력서 분석을 위한 비동기 큐(Message Queue) 도입 검토.
